# 提供商健康检查与 API Key 验证

## 功能概述

提供商健康检查是 AI 短剧工作室 v2.1 版本新增的诊断工具，用于：

- **检测 API 连接** - 验证提供商是否可访问
- **验证 API Key** - 测试 API Key 是否有效
- **诊断问题** - 快速识别连接和认证问题
- **性能监测** - 记录 API 响应时间

## 使用方式

### 1. 访问健康检查面板

打开模型配置弹窗 (⚙️)，点击 **"健康检查"** 标签页：

```
模型配置
├─ 全局配置
├─ API 提供商  
├─ 健康检查 ← 点击这里
├─ 对话模型
├─ 图片模型
└─ 视频模型
```

### 2. 检查单个提供商

#### 方式 A: 快速检查
1. 找到要检查的提供商
2. 点击右侧的 ⚡ (闪电图标)
3. 等待检查结果

#### 方式 B: 展开详情
1. 点击提供商卡片上的 ▼ (下拉箭头)
2. 查看完整的健康检查结果
3. 点击各模型的 "验证" 按钮测试具体模型

### 3. 批量检查所有提供商

点击右上角的 **"全部检查"** 按钮，系统会：
- 逐个检查所有配置的提供商
- 显示每个提供商的连接状态
- 返回整体摘要 (例如 "5 个提供商中 4 个连接正常")

## 检查结果解释

### 状态类型

#### ✅ 连接正常 (Healthy)
- **含义**: API 连接成功，API Key 有效
- **响应时间**: 显示毫秒数或秒数
- **操作**: 无需处理，继续使用

#### ⚠️ API Key 问题 (Invalid Key)
- **含义**: API Key 未配置或无效
- **原因**:
  - 全局 API Key 未设置
  - 提供商级 API Key 无效
  - 模型级 API Key 过期
- **解决方案**:
  1. 返回 "全局配置" 或 "API 提供商" 标签页
  2. 检查并重新配置 API Key
  3. 确保 API Key 有效且未过期

#### ❌ 连接失败 (Error)
- **常见原因**:
  - API 基础 URL 错误
  - 网络连接问题
  - API 服务器故障
  - 防火墙或代理阻止
- **解决方案**:
  1. 检查 API 基础 URL 是否正确
  2. 验证网络连接
  3. 确认 API 服务器状态
  4. 检查防火墙设置

#### ⏱️ 请求超时 (Timeout)
- **含义**: API 响应超过 10 秒
- **原因**:
  - API 服务器响应缓慢
  - 网络延迟高
  - 并发请求过多
- **解决方案**:
  - 稍后重试
  - 检查网络连接质量
  - 检查 API 配额限制

## 各提供商特定说明

### GitCC API
```
基础 URL: http://api.gitcc.com
支持模型: 所有内置模型
API Key: 必需
```

**健康检查会验证**:
- 服务器可访问性
- API Key 有效性
- 文本/图片/视频端点可用性

### OpenAI
```
基础 URL: https://api.openai.com/v1
支持模型: GPT-4o, GPT-4 Turbo, DALL-E 3, GPT-4o Realtime
API Key: 必需 (https://platform.openai.com/api-keys)
```

**常见问题**:
- 错误 401: API Key 无效或已删除
- 错误 429: 超过速率限制 (请升级账户)
- 错误 404: 模型名称错误

### Anthropic
```
基础 URL: https://api.anthropic.com
支持模型: Claude 3.5 Sonnet, Claude Opus 4.1
API Key: 必需 (https://console.anthropic.com)
```

**常见问题**:
- 错误 401: API Key 无效
- 错误 400: 请求格式错误

### DeepSeek
```
基础 URL: https://api.deepseek.com
支持模型: DeepSeek Chat
API Key: 必需 (https://platform.deepseek.com)
```

**优势**: 成本低，适合大文本处理

### Ollama (本地)
```
基础 URL: http://localhost:11434/v1
支持模型: Llama 2, Mistral
API Key: 可选 (本地模式)
```

**要求**:
- 本地安装 Ollama: https://ollama.ai
- 启动 Ollama 服务: `ollama serve`
- 拉取模型: `ollama pull llama2`

## 诊断流程

如果遇到问题，按以下步骤诊断：

### 步骤 1: 检查健康状态
1. 打开健康检查面板
2. 点击有问题提供商的 ⚡ 按钮
3. 查看返回的错误信息

### 步骤 2: 根据错误信息排查

#### 如果看到 "API Key 问题"
- 确保已在全局配置中设置 API Key
- 或在 API 提供商中为该提供商单独设置 API Key
- 验证 API Key 是否有效 (未过期)

#### 如果看到 "连接失败"
- 检查 API 基础 URL 是否正确
- 尝试在浏览器中访问该 URL
- 检查网络连接 (是否连接到互联网)
- 检查防火墙或代理设置

#### 如果看到 "请求超时"
- 检查网络延迟 (打开浏览器控制台查看)
- 稍后重试 (API 可能在处理其他请求)
- 检查 API 配额 (是否已达限制)

### 步骤 3: 验证模型

展开提供商详情，点击各模型的 "验证" 按钮：
- 测试特定模型是否可用
- 检查模型 API 名称是否正确
- 确认该模型是否在 API 配额内

## API 响应时间参考

| 提供商 | 典型响应时间 | 备注 |
|--------|------------|------|
| GitCC | 200-500ms | 国内服务器 |
| OpenAI | 500-2000ms | 可能有地域限制 |
| Anthropic | 300-1000ms | 支持流式响应 |
| DeepSeek | 300-1500ms | 成本最低 |
| Ollama | 100-500ms | 本地无网络延迟 |

响应时间超过 10 秒将被视为超时。

## 自动化诊断

### 周期性检查

建议定期检查提供商健康状态：
- **首次设置**: 配置后立即检查
- **日常使用**: 每周检查一次
- **生成失败**: 生成失败时检查连接

### 创建检查 Hook (可选)

如果安装了 Kiro，可以创建自动检查 Hook：

```json
{
  "name": "定期健康检查",
  "version": "1.0.0",
  "when": {
    "type": "userTriggered"
  },
  "then": {
    "type": "askAgent",
    "prompt": "运行所有提供商的健康检查"
  }
}
```

## 常见问题解答

### Q: 为什么总是显示 "API Key 问题"?
**A**: 检查以下项目:
1. 全局配置中是否设置了 API Key
2. 提供商配置中是否有专属 API Key
3. 模型配置中是否有模型级 API Key
4. API Key 是否有效 (未过期)

### Q: 如何测试自定义 API 提供商?
**A**: 
1. 返回 "API 提供商" 标签页
2. 添加自定义提供商 (输入基础 URL 和 API Key)
3. 返回 "健康检查" 标签页
4. 点击新提供商的 ⚡ 按钮

### Q: 健康检查失败，但生成却成功了?
**A**: 可能原因:
1. 检查时 API 服务暂时不可用，之后恢复
2. 某些 API 对测试请求有特殊限制
3. 生成请求和测试请求使用了不同的 API Key

### Q: Ollama 检查总是失败?
**A**: 确保:
1. Ollama 已安装并运行: `ollama serve`
2. 基础 URL 正确: `http://localhost:11434/v1`
3. 已拉取模型: `ollama pull llama2`
4. 本地防火墙允许 11434 端口

## 技术细节

### 测试逻辑

健康检查会对每个模型发送最小化请求：

**对话模型**:
```json
{
  "model": "gpt-4o",
  "messages": [{"role": "user", "content": "测试"}],
  "temperature": 0.1,
  "max_tokens": 10
}
```

**图片模型**:
```json
{
  "prompt": "测试图片生成",
  "size": "256x256"
}
```

**视频模型**:
```json
{
  "prompt": "测试视频生成",
  "duration": 4
}
```

### 超时设置

- 单个请求超时: 10 秒
- 全部检查总超时: 无限制 (间隔 300ms)

### 数据存储

- 检查结果仅保存在内存中
- 页面刷新后结果清空
- 支持离线查看最后检查时间
