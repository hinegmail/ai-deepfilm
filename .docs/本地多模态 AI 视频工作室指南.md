# 本地多模态 AI 视频工作室指南

本项目在设计上支持了标准的 **OpenAI 兼容协议**（如 `/v1/chat/completions` 等接口结构），因此无论是 Ollama 还是你自己用 Python 写的 API，都可以直接完美接入本系统。

下面是三大模块（文、图、视频）在本项目中的具体配置和调用方式：

---

### 1. 【文】文本创作与剧本生成（Ollama）

* **本地模型**：你安装在 Ollama 中的 `qwen`（通义千问）或 `gemma`。
* **接入原理**：Ollama 启动后，默认在本地 `http://127.0.0.1:11434` 运行，并且它**原生支持** OpenAI 兼容的 API 格式（即 `/v1/chat/completions`）。
* **在本项目中配置**：
  1. 打开本项目的**“模型配置”**界面。
  2. 新增一个**自定义 API 提供商 (Provider)**：
     * **提供商名称**：`Ollama-Local`
     * **API Base URL**：`http://127.0.0.1:11434/v1`
     * **API Key**：随便填写一个字符（如 `ollama`，本地服务不校验 Key，但前端组件要求必填）。
  3. 在该提供商下添加**对话模型（Chat Model）**：
     * **模型 ID**：填写你本地 Ollama 里的模型名称（例如 `qwen2.5:7b` 或 `gemma2:9b`，必须与 `ollama list` 显示的名称完全一致）。
  4. **使用**：在剧本创作、分镜导演 Prompt 优化时，选择该模型，即可完全在本地进行剧本分析和生成。

---

### 2. 【图】角色与概念图生成（Python + Ideogram 4）

* **本地模型**：通过 Python (`diffusers` 库) 加载的 `ideogram-4-nf4`。
* **接入原理**：通过我们在前面步骤中编写的 FastAPI 脚本，将 Ideogram 4 包装成符合 `/v1/images/generations` 规范的 API 服务（运行在本地 `http://127.0.0.1:8000`）。
* **在本项目中配置**：
  1. 在**“模型配置”**中新增提供商：
     * **API Base URL**：`http://127.0.0.1:8000/v1`
     * **API Key**：随便填写。
  2. 添加**图像模型（Image Model）**：
     * **模型 ID**：`ideogram-4`
  3. **使用**：在“角色设定”和“场景管理”中生成角色参考图和氛围图时，选择 `ideogram-4` 即可。

---

### 3. 【视频】镜头动画生成（Python + Sulphur 2 / LTX-Video）

* **本地模型**：通过 Python 加载的 Sulphur 2 蒸馏版或 FP8 混合精度的 `.safetensors`/`.gguf` 模型。
* **接入原理**：同样通过 Python FastAPI 写一个接收文本并调用 `LTXPipeline` 的接口（运行在本地另一个端口，如 `http://127.0.0.1:8001`）。
* **在本项目中配置**：
  1. 在**“模型配置”**中新增提供商：
     * **API Base URL**：`http://127.0.0.1:8001/v1`
     * **API Key**：随便填写。
  2. 添加**视频模型（Video Model）**：
     * **模型 ID**：`sulphur-2`
  3. **使用**：在“分镜导演”中，选择分镜点击“生成视频”时，选择 `sulphur-2`。系统会把生成的低分辨率基准视频返回，你可以在 Python 侧利用 4090 的算力进行 1080P 超分放大，最终将高清的 1080P mp4 视频呈现在软件界面中。

---

### 🚀 总结

通过这套组合拳，**你的 4090 显卡将作为本地最核心的算力中心**。你的整个短剧创作工作流（从剧本脑暴 ➡️ 角色和场景原画绘制 ➡️ 分镜视频生成 ➡️ 1080P 视频无损超分）都将在**不联网、不花钱、无限制**的本地环境中流畅运转。
