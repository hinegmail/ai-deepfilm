# Requirements Document

## Introduction

执行日志面板是一个实时显示AI生成操作日志的功能模块。当前系统在生成资产失败时，用户只能看到"生成失败"的提示，无法查看具体的错误原因。

该功能通过在侧边栏下方添加一个可折叠的日志面板，让用户能够：
- 实时查看所有生成操作的日志
- 快速诊断生成失败的原因
- 了解每个操作的执行情况

这将显著改善用户的调试体验。

## Glossary

- **日志面板** - 侧边栏下方显示生成操作日志的可折叠UI组件
- **渲染日志** - 记录单次AI生成操作的数据对象，包含操作类型、资源名称、状态、时间戳等
- **操作类型** - 生成操作的分类，包括：角色、服装变体、场景、关键帧、视频、脚本解析
- **资源名称** - 被生成对象的名称，如角色名、场景名等
- **状态** - 操作结果，成功（success）或失败（failed）
- **错误信息** - 操作失败时返回的错误描述

## Requirements

### Requirement 1: 面板基础显示

**User Story:** 作为一个创意创作者，我想在侧边栏看到执行日志面板，以便快速查看生成操作的状态。

#### Acceptance Criteria

1. 日志面板位于侧边栏下方，在"资产管理"菜单项后面
2. 面板标题为"执行日志"
3. 面板可以展开和折叠
4. 折叠时仅显示标题和展开/收起图标
5. 展开时显示日志列表

### Requirement 2: 实时日志显示

**User Story:** 作为一个用户，我想实时看到生成操作的日志，以便及时了解操作进展。

#### Acceptance Criteria

1. 新的生成操作日志应立即显示在面板中
2. 新日志显示在列表顶部（按时间倒序，最新优先）
3. 日志列表应自动滚动到最新条目
4. 日志在当前会话中保留

### Requirement 3: 日志内容显示

**User Story:** 作为一个用户，我想看到每条日志的关键信息，以便理解操作情况。

#### Acceptance Criteria

1. 每条日志显示：
   - 操作类型（中文标签，如"角色"、"场景"）
   - 资源名称
   - 操作状态（成功或失败）
   - 时间戳（HH:MM:SS）
   - 耗时（毫秒）
2. 成功操作显示绿色对勾图标和浅绿色背景
3. 失败操作显示红色警告图标和浅红色背景
4. 空列表时显示"暂无日志"提示

### Requirement 4: 错误信息展示

**User Story:** 作为一个遇到生成错误的用户，我想看到失败操作的错误详情，以便诊断问题原因。

#### Acceptance Criteria

1. 失败日志支持点击展开查看错误信息
2. 错误信息以代码字体显示
3. 错误信息支持换行和内部滚动
4. 展开的错误信息清晰显示 API 错误或异常描述

### Requirement 5: UI 风格一致性

**User Story:** 作为一个用户，我想日志面板风格与应用保持一致，以便获得统一的视觉体验。

#### Acceptance Criteria

1. 面板使用深色主题（slate-950 系列），与 Sidebar 协调
2. 成功显示绿色（emerald），失败显示红色（red）
3. 文本颜色满足 WCAG AA 对比度标准
4. 所有样式使用 Tailwind CSS
5. 边框、间距、圆角与现有设计保持一致
