# Task 3：集成测试与验证 - 测试指南

## 📋 任务目标

验证日志记录功能正确工作，并确保没有破坏现有功能。

## 🧪 测试方法

### 方法 A：使用浏览器控制台（推荐）

#### 1. 启动测试工具

打开浏览器的开发者工具（F12），在控制台中执行：

```javascript
LoggingTest.setup()
```

你会看到类似这样的输出：
```
✅ 日志收集已启动
现在执行生成操作（生成角色、场景、关键帧、视频等）
完成后，调用 getCollectedLogs() 或 printLogsReport() 查看结果
```

#### 2. 执行生成操作

按以下顺序执行生成操作：

**场景 A：角色和场景生成**
1. 在"场景角色"阶段生成一个角色
   - 选择一个角色 → 点击"生成角色色块" → 等待完成
   - 预期：日志中出现 `[日志记录] character`

2. 生成一个场景
   - 选择一个场景 → 点击"生成场景" → 等待完成
   - 预期：日志中出现 `[日志记录] scene`

3. 生成一个服装变体
   - 在角色的服装变体中 → 点击"生成变体图" → 等待完成
   - 预期：日志中出现 `[日志记录] character-variation`

**场景 B：关键帧生成**
1. 在"AI工作台"阶段
   - 选择一个镜头 → 点击"生成起始帧" → 等待完成
   - 预期：日志中出现 `[日志记录] keyframe`

2. 生成结束帧
   - 点击"生成结束帧" → 等待完成
   - 预期：日志中出现 `[日志记录] keyframe`

**场景 C：视频生成**
1. 在关键帧都生成后
   - 点击"生成视频" → 等待完成
   - 预期：日志中出现 `[日志记录] video`

#### 3. 查看日志报告

执行生成操作后，在控制台中执行：

```javascript
LoggingTest.report()
```

你会看到详细的日志收集报告，包括：
- 总日志数
- 按类型统计
- 按状态统计
- 详细日志列表
- 验证报告
- 验证得分

#### 4. 获取特定信息

```javascript
// 查看所有收集的日志
LoggingTest.getLogs()

// 查看特定类型的日志
LoggingTest.getByType('character')
LoggingTest.getByType('keyframe')

// 查看失败的日志
LoggingTest.getFailed()

// 查看成功的日志
LoggingTest.getSuccess()

// 导出为 JSON
LoggingTest.export()
```

#### 5. 停止测试

```javascript
LoggingTest.stop()
```

## 🧪 故障排查

### 如果没有看到日志记录

**问题 1：控制台中看不到 LoggingTest**
- 确保打开的是正确的浏览器标签
- 刷新页面后再试
- 检查浏览器控制台是否报错

**问题 2：执行生成操作后没有日志**
- 检查是否成功调用了 `LoggingTest.setup()`
- 等待生成操作真正完成（不要太快）
- 查看浏览器控制台是否有 JavaScript 错误

**问题 3：日志中只有部分类型的操作**
- 确保执行了所有类型的生成操作
- 一次只生成一个资源，等待完成后再做下一个

### 如果生成功能被破坏

**现象：点击生成按钮后没反应**
- 检查浏览器控制台是否有错误信息
- 检查 API Key 是否正确设置
- 尝试刷新页面

**现象：生成报错但没有错误提示**
- 这可能是日志记录异常
- 查看浏览器控制台的错误信息
- 检查 addRenderLog 是否被正确导入
