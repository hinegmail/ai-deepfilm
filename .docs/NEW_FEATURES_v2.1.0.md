## 🎯 核心新功能

### 1. 提供商健康检查
在模型配置中新增 **"健康检查"** 标签页，一键检查所有 API 提供商的连接状态。

**功能**:
- ✅ 检测 API 连接是否正常
- ✅ 验证 API Key 是否有效
- ✅ 测量 API 响应时间
- ✅ 自动诊断和分类错误

**使用方式**:
```
1. 点击 ⚙️ 模型配置
2. 选择 "健康检查" 标签页
3. 点击 "全部检查" 或个别 ⚡ 按钮
```

### 2. 智能错误诊断
系统会自动识别和分类各种错误，提供具体建议。

**识别的错误**:
| 错误 | 原因 | 建议 |
|------|------|------|
| ⚠️ API Key 问题 | API Key 无效/过期 | 在全局配置中重新设置 |
| ❌ 连接失败 | URL 错误或网络问题 | 检查 API 基础 URL 配置 |
| ⏱️ 请求超时 | 网络慢或 API 过载 | 稍后重试或检查网络 |

### 3. 模型级验证
支持逐个验证特定模型的 API Key 和可用性。

**功能**:
- 展开提供商查看所有模型
- 为每个模型单独测试
- 显示该模型的响应时间
- 返回具体的验证结果

### 4. 批量检查
"全部检查" 功能可同时检查所有配置的提供商。

**特性**:
- 自动间隔避免限流
- 实时显示进度
- 返回整体摘要
- 统计健康/错误/超时数量

### 检查能力
- ✅ 检测 API 可访问性

- ✅ 测试各模型端点
- ✅ 测量响应时间
- ✅ 识别错误类型
- ✅ 提供诊断建议

### 支持的提供商
| 提供商 | 内置 | 自定义 | 备注 |
|--------|------|--------|------|
| GitCC API | ✅ | - | 默认内置 |
| OpenAI | ✅ | ✅ | 推荐使用 |

| DeepSeek | ✅ | ✅ | 性价比高 |
| Ollama | ✅ | ✅ | 本地离线 |
| 其他 OpenAI 兼容 | - | ✅ | Together, Groq 等 |

### 错误诊断
系统能自动识别以下问题:
- 🔴 401/403 - API Key 认证失败
- 🔴 404 - API 端点不存在
- 🔴 429 - 速率限制 (限流)
- 🔴 500+ - 服务器错误
- 🟡 Timeout - 响应超时
- 🟡 Network - 网络连接问题
