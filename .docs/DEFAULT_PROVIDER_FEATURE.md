# 默认 API 提供商设置功能

## 功能说明

用户现在可以在"API 提供商"管理界面直接将任何提供商（包括内置和自定义）设置为**默认提供商**。

## 工作流程

### 1. 打开 API 提供商管理器

```
应用 → 模型配置 → API 提供商
```

### 2. 选择要设置为默认的提供商

在提供商卡片上悬停，会出现操作按钮：

**内置提供商**（上方区域）：
- ⭐ 星形按钮 - 设置为默认（未设置时显示）
- ✏️ 编辑按钮 - 编辑 API Key

**自定义提供商**（下方区域）：
- ⭐ 星形按钮 - 设置为默认（未设置时显示）
- ✏️ 编辑按钮 - 编辑提供商信息
- 🗑️ 删除按钮 - 删除提供商

### 3. 设置默认提供商

点击⭐星形按钮：
- ✅ 自动取消其他提供商的默认状态
- ✅ 设置此提供商为默认
- ✅ 提供商卡片上显示"默认"标签
- ✅ 系统会立即刷新

## UI 体验

### 默认提供商标识

```
提供商名称 [⭐ 默认]
API URL
是否已配置 API Key
```

### 操作按钮

当鼠标悬停在提供商卡片上时：

**未设置为默认时**：
```
⭐ 设置为默认  ✏️ 编辑  🗑️ 删除
```

**已设置为默认时**：
```
✏️ 编辑  🗑️ 删除
```
（⭐ 按钮隐藏，因为已是默认）

## 实际应用

### 场景 1：新增自定义提供商

1. 添加自定义提供商（例如 Together AI）
2. 点击⭐设置为默认
3. 之后添加新模型时，默认选中 Together AI
4. 提高工作效率，不需要重复选择

### 场景 2：切换主要提供商

1. 原来使用 OpenAI 作为默认
2. 现在想改为本地 Ollama
3. 在 Ollama 卡片上点击⭐
4. ✅ 自动切换，其他提供商自动取消默认状态

### 场景 3：多提供商混合使用

1. 设置 OpenAI 为默认（文本模型优先用它）
2. 添加自定义提供商 vLLM（本地备选）
3. 添加模型时，OpenAI 是默认选项
4. 如需切换，只需在下拉菜单中选择

## 技术实现

### 代码变更

**ProviderManager.tsx**：
- 新增 `handleSetDefault()` 方法
- 自动取消其他提供商的默认状态
- UI 显示默认提供商标识和按钮

**AddModelForm.tsx**：
- 使用默认提供商作为初始值
- `const defaultProvider = allProviders.find(p => p.isDefault) || allProviders[0]`

### 数据持久化

默认提供商状态保存在：
```
localStorage → ai_manga_studio_model_registry → providers[].isDefault
```

自动在浏览器关闭后保留。

## 常见问题

### Q: 如何取消默认提供商？

A: 在 ProviderManager 中点击其他提供商的⭐按钮，会自动取消旧的默认状态。

### Q: 删除默认提供商会怎样？

A: 
1. 系统会检查是否是默认提供商
2. 删除后，会自动选择列表中的第一个提供商作为新默认
3. 下次添加模型时使用新的默认提供商

### Q: 能否没有默认提供商？

A: 不能。系统总是会维护至少一个默认提供商，确保用户体验一致。

### Q: 添加新提供商后需要手动设置默认吗？

A: 不需要。新添加的提供商只是加入列表，不会自动成为默认。用户可以在需要时点击⭐设置。

## 最佳实践

### 1. 为常用提供商设置为默认

如果 80% 的时间都用某个提供商，就把它设为默认。

### 2. 定期检查默认提供商

如果切换了主要 API 服务商，记得更新默认设置。

### 3. 合理使用多提供商

```
推荐方案：
- 默认提供商: 主要选择（如 OpenAI）
- 备选提供商: 成本优化（如 DeepSeek）
- 本地提供商: 隐私保护（如 Ollama）
```

## 使用提示

### 💡 快速切换提供商

如果频繁在两个提供商间切换，可以：
1. 定期修改默认提供商
2. 或在添加模型时手动选择

### 💡 保持一致的工作流

```
每次工作前 → 检查默认提供商 → 开始工作
```

这样可以避免误用错误的 API 服务。

### 💡 成本优化

```
测试和开发 → 用便宜的 DeepSeek（设为默认）
生产和重要 → 用稳定的 OpenAI（临时切换）
```
