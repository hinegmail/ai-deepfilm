# 自定义模型优先使用指南

## 🎯 问题说明

在健康检查中，系统会验证您添加的模型。问题是：

**之前的行为**:
- 健康检查总是使用内置的 AGNES 模型（AGNES 2.0 Flash、AGNES 2.0 Standard）
- 即使您配置了自己的模型（Agnes-2.0-Flash、Agnes Image 2.1 Flash 等）

**现在的行为**:
- 健康检查优先使用您自定义添加的模型
- 只有在没有自定义模型时，才使用内置模型

---

## ✅ 修复内容

### 修改的函数：`getPreferredTestModel()`

```typescript
// 修复前：硬编码使用内置模型
'agnes': ['agnes:agnes-2.0-flash', 'agnes:agnes-2.0-standard']

// 修复后：优先使用自定义模型
1. 首先查找非内置的自定义模型
2. 如果找到自定义模型，返回第一个
3. 如果没有自定义模型，才使用内置模型的优先级列表
```

### 优先级顺序

```
1️⃣ 自定义模型（用户添加的）
   └─ 您在"对话模型"等标签中添加的所有模型

2️⃣ 内置模型（系统默认）
   ├─ openai:gpt-4o
   ├─ anthropic:claude-sonnet
   ├─ agnes:agnes-2.0-flash
   └─ 等等
```

### 如何判断一个模型是自定义还是内置？

在代码中：
```typescript

// 内置模型
```

## 💡 为什么这样设计

### 自定义优先的好处

1. **反映真实使用** - 检查您实际要用的模型
2. **准确诊断** - 发现您的模型可能遇到的问题
3. **尊重用户配置** - 不强制使用内置模型
4. **灵活性** - 支持任何模型组合

### 内置兜底的意义

- 如果用户没有添加自定义模型，系统仍能正常工作
- 提供合理的默认行为
- 确保首次使用时也能验证

## ❓ 常见问题

### Q: 为什么我看到的模型名称不同？

**A**: 
- **内置模型**: 系统预定义的，如 "AGNES 2.0 Flash"
- **自定义模型**: 您添加的，如 "Agnes-2.0-Flash"

它们是不同的模型定义。

### Q: 如何确认是使用了我的自定义模型？

**A**: 
1. 打开浏览器开发者工具 (F12)
2. 在 Console 中查看日志信息
3. 或者在模型验证结果中查看使用的模型名称

### Q: 能否强制使用内置模型？

**A**: 
不需要。如果您不想使用自定义模型，可以：
1. 在模型配置中禁用自定义模型
2. 或者删除自定义模型
3. 系统会自动切换到内置模型

### Q: 这会影响实际的生成过程吗？

**A**: 
不会。这只影响健康检查中使用哪个模型进行验证。
实际生成时，您选择的模型由"对话模型"等标签中的选择决定。
