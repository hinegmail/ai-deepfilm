## 一、产品概述

### 1.1 产品定位

**AI 短剧工作室（Print Film / AI Manga Studio）** 是一款面向短剧、漫剧、动态漫画与影视分镜创作者的 **AI 一站式视觉生产工作台**。

产品的核心差异化在于 **Script-to-Asset-to-Keyframe 工作流**：将"故事创意→结构化剧本→视觉资产→关键帧→视频片段"的完整创作链条统一在一个应用内完成，而非割裂在多个工具中。

### 1.2 核心价值主张

| 维度 | 价值描述 |
|------|---------|
| **流程统一** | 从剧本脑暴到视频导出，全流程在一个应用内完成，无需在多个工具间反复切换和复制提示词 |
| **关键帧驱动** | 引入动画/影视制作中的关键帧理念，先画后动，通过起止帧控制构图和动作，解决 Text-to-Video 角色不一致问题 |
| **资产约束** | 画面生成参考角色定妆、服装变体和场景概念图，提升角色一致性与场景连续性 |
| **模型自由** | 支持任意 OpenAI 兼容 API 提供商，用户完全掌控 API Key 和模型选择，不绑定任何付费平台 |
| **本地优先** | 项目数据存储在浏览器 IndexedDB，API Key 本地保存，无后端依赖，隐私优先 |

### 1.3 产品形态

产品同时支持三种部署形态：

| 形态 | 技术 | 适用场景 |
|------|------|---------|
| **Web 应用** | Vite 开发/构建 | 在线使用、快速体验 |
| **桌面应用** | Electron + electron-builder | Windows/macOS 离线使用，内建 HTTP 代理 |
| **容器部署** | Docker + Nginx | 私有化部署、团队共用 |

## 二、目标用户分析

### 2.1 用户画像

| 用户类型 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| **短剧创作者** | 将故事大纲快速转化为可预览的分镜视频 | 快速出片、低成本验证创意 |
| **漫剧/动态漫画制作人** | 批量生成角色和场景图，制作动态漫画 | 角色一致性、画风统一 |
| **分镜设计师** | 从剧本生成结构化分镜列表 | 高效分镜规划、镜头拆分 |
| **影视概念创意团队** | 快速生成概念图和预览视频 | 原型验证、视觉资产整理 |
| **AI 创意爱好者** | 使用本地模型进行创作实验 | 零成本、隐私保护、完全离线 |

### 2.2 用户痛点与解决方案

| 用户痛点 | 本产品解决方案 |
|---------|-------------|
| Text-to-Video 难以控制角色和构图 | 关键帧驱动：先生成起止帧，再插值生成运动 |
| 多工具切换、反复复制提示词 | 全流程统一：剧本→资产→关键帧→视频在一个项目状态中推进 |
| AI 生成角色前后不一致 | 资产约束：生成时参考角色定妆图和服装变体 |
| 付费 API 平台绑定 | 任意 OpenAI 兼容提供商接入，支持本地 Ollama |
| 数据隐私担忧 | 本地 IndexedDB 存储，无后端服务，API Key 本地保存 |
| 模型配置复杂 | 健康检查系统、三级 API Key 优先级、默认提供商机制 |

## 三、功能矩阵

```
AI 短剧工作室
├── 项目管理
│   ├── 项目创建与列表
│   ├── 自动保存（IndexedDB）
│   └── 资产库（跨项目复用角色/场景）

│   ├── 智能剧本拆解（AI 文本模型）
│   ├── 分镜规划（按时长/节奏生成镜头列表）
│   ├── 视觉化翻译（文字→图像提示词）
│   ├── 手动编辑（角色/场景/镜头/台词）
│   └── 项目配置（时长/语言/风格/模型）
│
├── Phase 02: 场景角色
│   ├── 角色定妆图生成
│   ├── 服装变体管理（日常/战斗/礼服等）
│   ├── 场景概念图生成
│   ├── 图片上传（手动补充参考资产）
│   └── 批量生成
│
├── Phase 03: AI工作台
│   ├── 镜头网格管理
│   ├── 关键帧生成（起始帧/结束帧）
│   ├── 上下文参考（场景+角色+服装）
│   ├── 镜头拆分（长镜头→子镜头）
│   ├── 视频生成（关键帧间插值）
│   └── 渲染日志记录
│
├── Phase 04: 制片导出
│   ├── 时间轴可视化预览
│   ├── 视频逐段播放
│   ├── 渲染日志集中查看
│   └── 资产批量下载（ZIP 打包）
│
├── 资产管理
│   ├── 角色/服装提示词编辑
│   ├── 场景提示词编辑
│   ├── 关键帧提示词编辑
│   ├── 视频提示词编辑
│   └── 搜索与筛选
│
└── 模型配置系统
    ├── 全局配置（API Key 验证）
    ├── API 提供商管理（内置 5 个 + 自定义无限）
    ├── 健康检查（连接诊断/错误分类/响应时间）
    ├── 对话/图片/视频模型管理
    └── 三级 API Key 优先级（模型>提供商>全局）
```

### 3.2 功能成熟度评估

| 功能模块 | 成熟度 | 说明 |
|---------|--------|------|
| 剧情创作 | ★★★★☆ | AI 拆解+手动编辑，功能完整 |
| 场景角色 | ★★★★☆ | 批量生成+服装变体，满足核心需求 |
| AI工作台 | ★★★☆☆ | 关键帧驱动理念先进，但视频生成受模型能力制约 |
| 制片导出 | ★★★☆☆ | 基本导出功能完备，缺乏高级剪辑能力 |
| 资产管理 | ★★★★☆ | 集中管理+搜索筛选，实用性强 |
| 模型配置 | ★★★★★ | 多提供商+健康检查+三级Key，业界领先 |
| 新手引导 | ★★★☆☆ | 有 Onboarding 流程，可进一步优化 |

### 3.3 AI 能力矩阵

| AI 能力 | 用途 | 默认模型 | 可替换 |
|--------|------|---------|--------|
| 文本对话 | 剧本拆解、角色分析、提示词优化 | AGNES 2.0 Flash | 任意 OpenAI 兼容 |
| 图像生成 | 角色定妆、场景概念、关键帧 | Agnes Image 2.1 Flash | 任意 OpenAI 兼容 |
| 视频生成 | 关键帧间运动过渡 | Agnes-Video-V2.0 | Sora/Veo/豆包 Seedance/自定义 |

### 3.4 视频生成模式

产品适配了三种视频生成模式，体现了对行业生态的深度理解：

| 模式 | 适配模型 | 特点 |
|------|---------|------|
| **同步模式 (sync)** | Veo 系列 | 请求即返回，适合快速验证 |
| **异步模式 (async)** | Sora 系列 | 任务轮询机制，支持长视频生成 |
| **豆包模式 (doubao)** | 豆包 Seedance | 国内模型适配，支持远程图片 URL |

## 五、竞争分析

### 5.1 竞品对比

| 维度 | AI 短剧工作室 | 通用 AI 聊天工具 | 专业视频剪辑软件 | AI 视频生成平台 |
|------|-------------|----------------|---------------|--------------|
| 全流程覆盖 | ✅ 剧本→视频 | ❌ 仅文本 | ❌ 仅剪辑 | ❌ 仅视频 |
| 关键帧驱动 | ✅ | ❌ | ❌ | 部分 |
| 资产一致性 | ✅ 角色定妆约束 | ❌ | ❌ | 部分 |
| 多模型支持 | ✅ 任意兼容 API | 单一 | N/A | 单一 |
| 本地离线 | ✅ Ollama | ❌ | ✅ | ❌ |
| 数据隐私 | ✅ 本地存储 | ❌ | ✅ | ❌ |
| 部署灵活 | ✅ Web/桌面/Docker | Web | 桌面 | Web |
| 使用门槛 | 中 | 低 | 高 | 低 |
| 成本 | 免费(模型费自付) | 低 | 高 | 中-高 |

### 5.2 核心竞争壁垒

1. **流程一体化** — 市面上少有将剧本创作、视觉资产、关键帧、视频生成统一在一个工作台的产品
2. **关键帧驱动理念** — 借鉴动画/影视制作的专业方法论，比纯 Text-to-Video 更可控
3. **模型自由度** — 不绑定任何单一 AI 平台，用户可自由切换和组合
4. **本地隐私优先** — 数据完全在本地，适合对隐私敏感的用户和企业

### 5.3 竞争劣势

1. **使用门槛** — 需要用户理解关键帧、提示词等概念，学习曲线较陡
2. **PC 端限制** — 明确不支持移动端，缩小了用户群体
3. **视频质量依赖模型** — 产品本身不包含视频生成能力，效果取决于所选 API 模型
4. **缺乏协作功能** — 纯本地应用，不支持多人协作

## 十、总结

AI 短剧工作室是一款 **有明确差异化定位** 的 AI 创作工具。其核心优势在于：

1. **流程一体化** — 覆盖从剧本到视频的完整创作链条
2. **关键帧驱动** — 解决 AI 视频生成的控制力难题
3. **模型自由度** — 不绑定任何平台，用户完全掌控
4. **隐私优先** — 本地存储，无后端依赖

当前产品在 **模型配置系统** 和 **工作流设计** 上已达到较高成熟度，但在 **视频生成体验**（等待时间长、质量依赖模型）和 **数据安全**（缺乏备份机制）方面仍有提升空间。

建议下一阶段重点推进 **项目备份/恢复**、**视频生成体验优化** 和 **角色一致性增强**，同时探索 **企业版** 和 **增值服务** 的商业化路径。
